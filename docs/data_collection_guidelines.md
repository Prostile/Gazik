# Сбор реальных LAS и геологических материалов

Минимум для Hybrid #2 — исходный LAS с `GR`, `RHOB`, `NPHI`, `DT`, `RT` и глубиной.
Файл не пересчитывают и не переименовывают: исходные мнемоники и единицы фиксируют в
`well_metadata.yaml`. Также желательно сохранять отчет, tops, керн, шлам,
стратиграфию и интерпретацию с источником и уверенностью.

Структура и рабочий пример находятся в `data/raw_las/README.md`. Tops имеют поля
`well_id,top,base,name,type,confidence`; литология —
`well_id,top,base,lithology,facies,source,confidence`. Границы задаются в единицах
исходного depth; `top < base`, пропуски и сомнительные интервалы описывают в
`data_quality.comments`.

Обязательно фиксируются источник, лицензия и разрешение на распространение. Наличие
ручной разметки пока не обязательно, но существующую разметку нельзя отбрасывать.

```powershell
python -m synthetic_well_logs validate-dataset-structure --root data/raw_las
```
